Build the package and upload to PyPI to reserve the "cyrus" name.

**Step 1: Install build tools** (in the dev environment):
```bash
pip install build twine
```

**Step 2: Build the distribution:**
```bash
python -m build
```

This should produce:
- `dist/cyrus-0.0.0-py3-none-any.whl`
- `dist/cyrus-0.0.0.tar.gz`

**Step 3: Verify the build** before uploading:
```bash
twine check dist/*
```
Both files must pass. If either fails, fix pyproject.toml metadata and rebuild.

**Step 4: Upload to PyPI:**
```bash
twine upload dist/*
```

This will prompt for credentials. Use:
- Username: `__token__`
- Password: the PyPI API token (from `PYPI_API_TOKEN` env var or entered manually)

Alternatively, if the user has configured `~/.pypirc` or the `TWINE_USERNAME`/`TWINE_PASSWORD` env vars, `twine upload` will use those automatically.

**Step 5: Verify the package is live:**
```bash
pip install cyrus==0.0.0 --dry-run
```
Or check: `https://pypi.org/project/cyrus/0.0.0/`

If the upload fails because the name is taken (unlikely — verified available 2026-04-11), immediately escalate. Do NOT choose an alternative name without user approval.

**Step 6: Add dist/ to .gitignore** if not already present, and do NOT commit the dist/ directory.
